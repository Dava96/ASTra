## 1. Executive Summary

**ASTra (Autonomous System for Technical Reasoning & Action)** is a self-hosted, domain-agnostic AI Coding Agent designed for high-autonomy software development. Unlike simple "chat-with-code" tools, ASTra implements a **Plan-Act-Reflect** cognitive architecture that allows it to:
1.  **Ingest** entire codebases into a semantic Knowledge Graph.
2.  **Plan** complex refactors using a dedicated planner/critic loop.
3.  **Execute** changes via robust tooling (Aider, Git, Shell).
4.  **Verify** outcomes through automated testing and self-healing.

## 2. Verified Architecture

The system follows a Hexagonal (Ports & Adapters) Architecture to ensure domain agnosticism and testability.

### 2.1 Core Orchestrator (`astra/core`)
The brain of the system, responsible for task lifecycle management.
-   **Cognitive Loop**: Implements a `Plan -> Execute -> Test -> Finalize` state machine.
-   **Critic-Refinement**: Uses a dual-LLM approach (Planner vs. Critic) to refine implementation plans before writing code.
-   **Task Queue**: Asynchronous, priority-based task management supporting concurrent operations.
-   **Context Gathering**: Dynamically retrieves context using both Vector Search (ChromaDB) and Knowledge Graph (NetworkX) traversal.

### 2.2 Ingestion Engine (`astra/ingestion`)
A high-performance codebase indexer optimized for speed and structure.
-   **Parallel Processing**: Uses `ProcessPoolExecutor` to parse files concurrently.
-   **O(1) Incremental Updates**: Uses a content-hash cache to skip unchanged files instantly (0.01s scan time).
-   **Dependency Resolution**: Builds a real graph of imports (`A -> imports -> B`), enabling "impact analysis" (e.g., "If I change B, what breaks?").

### 2.3 Ports & Adapters (`astra/adapters`)
Decouples core logic from external interfaces.
-   **Gateways**:
    -   **Discord**: Full command-and-control bot interface (`/feature`, `/fix`, `/status`). Supports rich UI interactions (buttons, embeddings).
    -   **Console**: CLI interface for local, headless operation.
-   **Storage**:
    -   **ChromaDB**: Vector storage for semantic code search.
    -   **KnowledgeGraph**: GraphML-based structure for relationship tracking.
